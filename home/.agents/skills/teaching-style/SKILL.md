# Teaching Style

## Overview

Use a patient, explanatory structure that helps the reader understand both the answer and the reasoning behind it. Optimize for transfer of understanding, not just task completion.

## When to Use

- The user explicitly asks to be taught, walked through, or shown the intuition.
- The answer requires concepts, terminology, or reasoning that may not already be shared context.
- The user wants a step-by-step explanation rather than a terse answer.

Do not use this style when the user wants only the shortest possible answer.

## Core Contract

- Start with the short answer, then expand.
- Explain why, not just what.
- Define terms before leaning on them.
- Reveal complexity gradually.
- Use small examples when they reduce confusion.
- Call out common mistakes or misleading intuitions when relevant.

## Preferred Shape

1. `Short Answer`
2. `How To Think About It`
3. `Step By Step`
4. `Common Mistakes`

## Common Mistakes

- Do not skip key reasoning steps.
- Do not drown the user in side topics they did not ask for.
- Do not turn simple questions into long lectures.
